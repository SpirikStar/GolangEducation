## 1. Введение: зачем изучать сортировки и поиск

**Сортировка** — упорядочивание элементов последовательности по некоторому ключу (число, строка, составной критерий). **Поиск** — нахождение элемента с заданным свойством или позиции для вставки в уже упорядоченную структуру.

В реальных системах:

- упорядоченные данные позволяют применять **бинарный поиск** и объединять отсортированные потоки;
- многие алгоритмы на графах и строках опираются на приоритетные очереди и сортировку рёбер/суффиксов;
- понимание **памяти и кэша** помогает выбрать не «самый красивый на бумаге», а приемлемый на практике алгоритм.

Go как язык со **статической типизацией** и **сборкой мусора** накладывает типичные ограничения: лишние аллокации слайсов в цикле дороги; копирование больших структур лучше избегать; для горячих путей используют `sort.Slice` с замыканиями или типы, реализующие `sort.Interface`.

## 2. Асимптотика и устойчивость

### 2.1. Нотация O, Θ, Ω (интуитивно)

Пусть \(n\) — размер входа (число элементов).

- **O(f(n))** — верхняя оценка роста времени (или памяти): «не хуже чем константа × f(n) при больших n».
- **Ω(f(n))** — нижняя оценка (алгоритм не может быть быстрее этого порядка в худшем случае для класса задач).
- **Θ(f(n))** — «плотная» оценка: и сверху, и снизу того же порядка.

На практике чаще всего говорят «алгоритм работает за O(n log n)», имея в виду **худший** или **амортизированный** случай — уточнять нужно всегда.

### 2.2. Сравнения и нижняя граница

Если алгоритм сортировки основан только на **попарных сравнениях** и не использует структуру ключей (например, что ключи — целые в ограниченном диапазоне), то в худшем случае нужно **Ω(n log n)** сравнений. Отсюда: merge sort и heap sort достигают O(n log n) по времени в худшем случае; быстрая сортировка в среднем часто O(n log n), но без доработок худший случай O(n²).

### 2.3. Устойчивость (stable)

**Устойчивая** сортировка сохраняет относительный порядок **равных** ключей. Это важно при сортировке по вторичному ключу: сначала по фамилии, затем по имени — после второй сортировки по имени порядок по фамилии для одинаковых имён должен сохраниться.

В Go:

- `sort.SliceStable`, `sort.Stable` — устойчивые;
- `sort.Slice`, `sort.Sort` для типов с `Less` — в общем случае **не гарантируют** устойчивость (зависит от реализации; для стандартных типов смотрите документацию к конкретной функции).

### 2.4. Память

- **In-place** — O(1) дополнительной памяти (или O(log n) под стек рекурсии), перестановки в том же массиве.
- **Слияние** обычно требует O(n) дополнительной памяти для буфера.

## 3. Представление данных в Go

### 3.1. Массив и слайс

```go
// массив — значение фиксированной длины, копируется при присваивании
var a [5]int

// слайс — дескриптор (указатель, len, cap) над массивом
s := []int{3, 1, 4, 1, 5}
```

**По шагам (что здесь происходит).**

1. `var a [5]int` — выделили массив из пяти нулей: `[0 0 0 0 0]` (длина всегда 5).
2. `s := []int{3, 1, 4, 1, 5}` — создали слайс длины 5: индексы `0..4`, значения как в фигурных скобках.

**Результат в памяти (для `s`).** Длина `len(s) == 5`, при печати: `[3 1 4 1 5]`.

**Пример использования (печать и срез).**

```go
package main

import "fmt"

func main() {
	s := []int{3, 1, 4, 1, 5}
	fmt.Println("весь слайс:", s)           // весь слайс: [3 1 4 1 5]
	fmt.Println("элемент с индексом 2:", s[2]) // элемент с индексом 2: 4
	fmt.Println("кусок [1:4):", s[1:4])     // кусок [1:4): [1 4 1]  — правый индекс 4 не входит
}
```

Алгоритмы чаще формулируют для **индексов** `[left, right)` — полуинтервал: левый включён, правый исключён. Так проще избежать off-by-one ошибок.

### 3.2. Интерфейс сравнения

Для обобщённой сортировки «что меньше» удобно передавать функцию:

```go
type LessFunc[T any] func(a, b T) bool

// Пример: сортировка по возрастанию int
lessInt := func(a, b int) bool { return a < b }
```

**По шагам.** Функция `lessInt(2, 5)` сравнивает два числа: `2 < 5` → `true` (первое «меньше» в смысле сортировки по возрастанию). `lessInt(5, 2)` → `false`.

**Результат.** Это не печать, а правило: при таком `Less` «меньший» элемент идёт раньше в отсортированном порядке.

**Пример использования (ручная проверка пары чисел).**

```go
package main

import "fmt"

func main() {
	lessInt := func(a, b int) bool { return a < b }
	fmt.Println(lessInt(2, 5)) // true
	fmt.Println(lessInt(5, 2)) // false
}
```

В учебных примерах ниже часто используется `[]int` и сравнение `a < b` для краткости.

### 3.3. Обмен элементами

```go
func swap[T any](a []T, i, j int) {
	a[i], a[j] = a[j], a[i]
}
```

**По шагам.** Пусть `a = [10, 20, 30]`, вызов `swap(a, 0, 2)`: сначала читаем `a[0]` и `a[2]`, затем записываем наоборот → `[30, 20, 10]`.

**Результат после вызова.** `fmt.Println(a)` → `[30 20 10]`.

**Пример использования.**

```go
package main

import "fmt"

func swapInt(a []int, i, j int) { a[i], a[j] = a[j], a[i] }

func main() {
	x := []int{10, 20, 30}
	swapInt(x, 0, 2)
	fmt.Println(x) // [30 20 10]
}
```

Для больших структур `T` обмен по значению может копировать много байт; тогда сортируют **индексы** или **указатели** `[]*Item`.

## 4. Сортировка вставками (Insertion Sort)

**Идея.** Поддерживаем инвариант: префикс `[0, i)` уже отсортирован. Элемент `a[i]` «вставляем» на правильную позицию сдвигом вправо больших элементов.

**Сложность.**

- Худший случай (массив в обратном порядке): **O(n²)** сравнений и сдвигов.
- Лучший (уже отсортирован): **O(n)** сравнений, почти без сдвигов.
- Почти отсортированные данные — сильный плюс вставкам.

**Устойчивость.** Да (при вставке «вправо» для равных не переставляются лишний раз).

**Применение в Go.** Внутри `sort` пакета вставками сортируют **короткие** отрезки как оптимизацию (малый n — низкая константа).

### Пример: сортировка вставками для `[]int`

```go
package main

import "fmt"

// InsertionSort сортирует слайс по неубыванию.
func InsertionSort(a []int) {
	for i := 1; i < len(a); i++ {
		key := a[i]
		j := i - 1
		for j >= 0 && a[j] > key {
			a[j+1] = a[j]
			j--
		}
		a[j+1] = key
	}
}

func main() {
	x := []int{5, 2, 4, 6, 1, 3}
	InsertionSort(x)
	fmt.Println(x) // [1 2 3 4 5 6]
}
```

**По шагам (массив короче: `[5, 2, 4, 1]` — так проще проследить руками).**

Инвариант: отрезок с индексами `[0, i)` уже отсортирован. На каждом шаге вставляем `a[i]`.

| Шаг | i | key (=a[i] до сдвигов) | Массив до вставки | Что делаем коротко | Массив после |
|-----|---|-------------------------|-------------------|--------------------|--------------|
| старт | 1 | 2 | `[5, 2, 4, 1]` | 5 > 2 → сдвинуть 5 вправо, вставить 2 в позицию 0 | `[2, 5, 4, 1]` |
| 2 | 2 | 4 | `[2, 5, 4, 1]` | 5 > 4 → сдвинуть 5; 2 не > 4 → вставить 4 | `[2, 4, 5, 1]` |
| 3 | 3 | 1 | `[2, 4, 5, 1]` | сдвигаем 5, 4, 2 вправо, вставляем 1 в начало | `[1, 2, 4, 5]` |

Для полного примера из кода `[5, 2, 4, 6, 1, 3]` логика та же: каждый новый элемент «протаскивается» влево, пока слева не окажется меньше или равный.

**Результат при запуске `main` из листинга выше (вывод в консоль).**

```text
[1 2 3 4 5 6]
```

**Пример использования в другом месте программы.**

```go
scores := []int{88, 71, 95, 71}
InsertionSort(scores)
// scores теперь [71 71 88 95] — сортировка по неубыванию, на месте (тот же слайс)
```

Важно: функция **меняет переданный слайс**; копию не делает. Если нужен исходный порядок — перед вызовом сделайте `copy` или `slices.Clone` (Go 1.21+).

### Упражнение в коде: обобщение на `Less`

Перепишите `InsertionSort` так, чтобы принимать `less func(i, j int) bool`, сравнивая элементы по индексам (полезно, когда элементы — структуры и сравнение нестандартное).

## 5. Сортировка выбором (Selection Sort)

**Идея.** На шаге `i` находим минимум в `a[i:]` и меняем с `a[i]`.

**Сложность.** Всегда **O(n²)** сравнений; перестановок O(n).

**Устойчивость.** Нет (обмен может перепрыгнуть равные элементы).

**Зачем знать.** Исторически и для контраста с вставками; на практике редко используется как основной метод.

```go
func SelectionSort(a []int) {
	n := len(a)
	for i := 0; i < n-1; i++ {
		minIdx := i
		for j := i + 1; j < n; j++ {
			if a[j] < a[minIdx] {
				minIdx = j
			}
		}
		if minIdx != i {
			a[i], a[minIdx] = a[minIdx], a[i]
		}
	}
}
```

**По шагам (массив `[4, 2, 7, 1]`).** На шаге `i` ищем **минимум** среди `a[i], a[i+1], …` и меняем местами с `a[i]`.

| i | Ищем минимум в | Найденный индекс | Обмен | Массив после |
|---|-----------------|------------------|--------|--------------|
| 0 | `[4, 2, 7, 1]` | индекс 3, значение 1 | `a[0]` ↔ `a[3]` | `[1, 2, 7, 4]` |
| 1 | `[2, 7, 4]` | индекс 1, значение 2 | без обмена | `[1, 2, 7, 4]` |
| 2 | `[7, 4]` | индекс 3, значение 4 | `a[2]` ↔ `a[3]` | `[1, 2, 4, 7]` |

Шаг `i = 3` не нужен: последний элемент уже на месте.

**Результат.** После `SelectionSort` для `[]int{4, 2, 7, 1}` слайс станет `[1 2 4 7]`.

**Пример использования (один файл — можно сразу `go run`).**

```go
package main

import "fmt"

func SelectionSort(a []int) {
	n := len(a)
	for i := 0; i < n-1; i++ {
		minIdx := i
		for j := i + 1; j < n; j++ {
			if a[j] < a[minIdx] {
				minIdx = j
			}
		}
		if minIdx != i {
			a[i], a[minIdx] = a[minIdx], a[i]
		}
	}
}

func main() {
	data := []int{4, 2, 7, 1}
	SelectionSort(data)
	fmt.Println(data) // [1 2 4 7]
}
```

## 6. Пузырьковая сортировка (Bubble Sort)

**Идея.** Повторно проходим массив, «всплывая» больший элемент к концу.

**Сложность.** **O(n²)**; можно раннее прерывание, если за проход не было обменов (лучший случай O(n)).

**Устойчивость.** Да, если при `>` обмениваемся (равные не трогаем).

```go
func BubbleSort(a []int) {
	n := len(a)
	for i := 0; i < n; i++ {
		swapped := false
		for j := 0; j < n-1-i; j++ {
			if a[j] > a[j+1] {
				a[j], a[j+1] = a[j+1], a[j]
				swapped = true
			}
		}
		if !swapped {
			break
		}
	}
}
```

**По шагам (массив `[3, 1, 2]`).** Внутренний цикл сравнивает соседей `j` и `j+1`; если слева больше — меняем. После каждого **полного прохода** самый большой элемент «всплывает» к концу отрезка.

| Проход (i) | Пары (j, j+1) | Обмены? | Массив после прохода |
|------------|----------------|---------|----------------------|
| 0 | (3,1), (1,2) | 3>1 да; 3>2 да | `[1 2 3]` |
| 1 | уже без обменов | swapped=false, **выход** из внешнего цикла | `[1 2 3]` |

Для уже отсортированного `[1, 2, 3]` первый же проход даст `swapped == false` и алгоритм остановится быстро (лучший случай по времени).

**Результат.** `BubbleSort([]int{3, 1, 2})` → слайс `[1 2 3]`.

**Пример использования.**

```go
package main

import "fmt"

func BubbleSort(a []int) {
	n := len(a)
	for i := 0; i < n; i++ {
		swapped := false
		for j := 0; j < n-1-i; j++ {
			if a[j] > a[j+1] {
				a[j], a[j+1] = a[j+1], a[j]
				swapped = true
			}
		}
		if !swapped {
			break
		}
	}
}

func main() {
	x := []int{3, 1, 2}
	BubbleSort(x)
	fmt.Println(x) // [1 2 3]
}
```

**Практика.** Пузырёк учат как учебный пример; в продакшене предпочитают вставки на малых n или `sort`.

## 7. Сортировка Шелла

**Идея.** Обобщение вставок: сначала сортируем подпоследовательности с большим **шагом** `gap`, постепенно уменьшая `gap` до 1. Уменьшает число «дальних» инверсий.

**Сложность.** Зависит от последовательности шагов; строгий анализ нетривиален. На практике лучше квадратичных простых методов на средних n.

```go
func ShellSort(a []int) {
	n := len(a)
	// последовательность gaps (пример; есть более оптимальные схемы)
	for gap := n / 2; gap > 0; gap /= 2 {
		for i := gap; i < n; i++ {
			temp := a[i]
			j := i
			for j >= gap && a[j-gap] > temp {
				a[j] = a[j-gap]
				j -= gap
			}
			a[j] = temp
		}
	}
}
```

**По шагам (очень маленький пример `n = 4`, массив `[8, 3, 5, 2]`).** Сначала `gap = 2`: сортируются подпоследовательности с шагом 2 (индексы 0,2 и 1,3). Затем `gap = 1` — это уже обычные вставки по всему массиву.

Упрощённо для одного `gap = 2` (идея: каждый элемент тянется к началу с шагом `gap`):

- Индексы с шагом 2: позиции 0 и 2 → значения 8 и 5; после вставочного шага для этого «столбца» порядок улучшается.
- Индексы 1 и 3 → 3 и 2.

Точная последовательность зависит от реализации цикла; студентам полезно поставить `fmt.Println` внутрь цикла и **распечатать** массив после каждого `gap`.

**Результат.** После полного `ShellSort` массив `[8, 3, 5, 2]` станет отсортированным по неубыванию: `[2 3 5 8]`.

**Пример использования.**

```go
package main

import "fmt"

func ShellSort(a []int) {
	n := len(a)
	for gap := n / 2; gap > 0; gap /= 2 {
		for i := gap; i < n; i++ {
			temp := a[i]
			j := i
			for j >= gap && a[j-gap] > temp {
				a[j] = a[j-gap]
				j -= gap
			}
			a[j] = temp
		}
	}
}

func main() {
	x := []int{8, 3, 5, 2}
	ShellSort(x)
	fmt.Println(x) // [2 3 5 8]
}
```

## 8. Быстрая сортировка (Quick Sort)

**Идея.** Выбираем **опорный** элемент (pivot), **разделяем** массив на части `< pivot`, `== pivot`, `> pivot` (или два подмассива в in-place схеме Lomuto/Hoare), рекурсивно сортируем части.

**Среднее время.** O(n log n) при «хороших» разбиениях.

**Худшее время.** O(n²), если pivot всегда минимум или максимум (уже отсортированный массив и наивный выбор края).

**Улучшения.**

- **Случайный pivot** или **median-of-three**;
- при малом размере подмассива переключаться на **вставки**;
- **introsort** — ограничение глубины рекурсии с переходом на heap sort (идея используется в стандартных библиотеках C++ и др.).

### Пример: разбиение Lomuto (учебный)

```go
func partitionLomuto(a []int, lo, hi int) int {
	pivot := a[hi]
	i := lo
	for j := lo; j < hi; j++ {
		if a[j] <= pivot {
			a[i], a[j] = a[j], a[i]
			i++
		}
	}
	a[i], a[hi] = a[hi], a[i]
	return i
}

func quickSort(a []int, lo, hi int) {
	if lo >= hi {
		return
	}
	// порог для вставок (типично 10–32)
	const cutoff = 12
	if hi-lo+1 <= cutoff {
		insertionSortRange(a, lo, hi)
		return
	}
	p := partitionLomuto(a, lo, hi)
	quickSort(a, lo, p-1)
	quickSort(a, p+1, hi)
}

func insertionSortRange(a []int, lo, hi int) {
	for i := lo + 1; i <= hi; i++ {
		key := a[i]
		j := i - 1
		for j >= lo && a[j] > key {
			a[j+1] = a[j]
			j--
		}
		a[j+1] = key
	}
}

func QuickSort(a []int) {
	if len(a) == 0 {
		return
	}
	quickSort(a, 0, len(a)-1)
}
```

**По шагам: только разбиение Lomuto (один вызов `partitionLomuto`).**

Возьмём отрезок `a[lo:hi+1]` = `[4, 2, 7, 1, 3]`, индексы `lo=0`, `hi=4`. **Опорный** (pivot) — последний элемент: `pivot = a[4] = 3`. Переменная `i` растёт: всё, что `≤ pivot`, собираем в левую часть.

| j | a[j] | Условие a[j] <= pivot | Действие | Массив после шага j | i |
|---|------|------------------------|----------|---------------------|---|
| — | — | старт | i=0 | `[4, 2, 7, 1, 3]` | 0 |
| 0 | 4 | 4 <= 3? нет | без обмена | `[4, 2, 7, 1, 3]` | 0 |
| 1 | 2 | да | swap(0,1), i++ | `[2, 4, 7, 1, 3]` | 1 |
| 2 | 7 | нет | — | `[2, 4, 7, 1, 3]` | 1 |
| 3 | 1 | да | swap(1,3), i++ | `[2, 1, 7, 4, 3]` | 2 |

После цикла ставим pivot на «границу»: обмен `a[i]` и `a[hi]` → `[2, 1, 3, 4, 7]`. Индекс возврата `p = 2`: слева от `3` — элементы `≤ 3`, справа — `> 3` (в этой схеме нестрого слева). Дальше рекурсивно сортируют `[2,1]` и `[4,7]`.

**Результат после полного `QuickSort`.** Например, `QuickSort([]int{5, 2, 4, 1, 3})` даст `[1 2 3 4 5]`.

**Пример использования.** Скопируйте в один файл все функции из листинга (`partitionLomuto`, `quickSort`, `insertionSortRange`, `QuickSort`) и добавьте:

```go
package main

import "fmt"

// ... все функции QuickSort из блока выше ...

func main() {
	x := []int{5, 2, 4, 1, 3}
	QuickSort(x)
	fmt.Println(x) // [1 2 3 4 5]
}
```

**Замечание по устойчивости.** Классическая in-place быстрая сортировка **не устойчива**.

## 9. Сортировка слиянием (Merge Sort)

**Идея.** Разделить пополам, отсортировать рекурсивно, **слить** два отсортированных массива за O(n).

**Время.** **O(n log n)** в худшем случае.

**Память.** O(n) на временный буфер (можно один буфер переиспользовать).

**Устойчивость.** Да, если при слиянии при равенстве брать сначала элемент из левой половины.

```go
func merge(dst, left, right []int) {
	i, j, k := 0, 0, 0
	for i < len(left) && j < len(right) {
		if left[i] <= right[j] { // <= даёт устойчивость
			dst[k] = left[i]
			i++
		} else {
			dst[k] = right[j]
			j++
		}
		k++
	}
	for i < len(left) {
		dst[k] = left[i]
		i++
		k++
	}
	for j < len(right) {
		dst[k] = right[j]
		j++
		k++
	}
}

func mergeSortBuf(a, buf []int, lo, hi int) {
	// [lo, hi) полуинтервал
	if hi-lo <= 1 {
		return
	}
	mid := lo + (hi-lo)/2
	mergeSortBuf(a, buf, lo, mid)
	mergeSortBuf(a, buf, mid, hi)

	// сливаем a[lo:mid] и a[mid:hi] в buf, затем копируем назад
	left := a[lo:mid]
	right := a[mid:hi]
	tmp := buf[lo:hi]
	merge(tmp, left, right)
	copy(a[lo:hi], tmp)
}

func MergeSort(a []int) {
	if len(a) <= 1 {
		return
	}
	buf := make([]int, len(a))
	mergeSortBuf(a, buf, 0, len(a))
}
```

**По шагам: только слияние двух уже отсортированных кусков.**

`left = [1, 4, 6]`, `right = [2, 5]`. В `dst` пишем по одному наименьшему из голов `left` и `right`.

| Шаг | Сравниваем | Кладём в dst | Остаток left | Остаток right |
|-----|------------|--------------|--------------|---------------|
| 1 | 1 и 2 | 1 | [4, 6] | [2, 5] |
| 2 | 4 и 2 | 2 | [4, 6] | [5] |
| 3 | 4 и 5 | 4 | [6] | [5] |
| 4 | 6 и 5 | 5 | [6] | [] |
| 5 | — | 6 | [] | [] |

Итог `dst`: `[1, 2, 4, 5, 6]`.

**Результат после `MergeSort`.** `MergeSort([]int{5, 2, 8, 1})` → `[1 2 5 8]`.

**Пример использования.** Объедините в одном файле функции `merge`, `mergeSortBuf`, `MergeSort` из листинга и вызовите:

```go
package main

import "fmt"

// merge, mergeSortBuf, MergeSort — как в листинге выше

func main() {
	data := []int{5, 2, 8, 1}
	MergeSort(data)
	fmt.Println(data) // [1 2 5 8]
}
```

**Когда выбирать.** Нужна **предсказуемая** O(n log n), внешняя сортировка больших данных, связные списки — merge sort естественен.

## 10. Пирамидальная сортировка (Heap Sort)

**Идея.** Строим **бинарную кучу** (max-heap): родитель ≥ детей. Максимум в корне; меняем с концом, уменьшаем кучу, просеиваем корень.

**Время.** O(n log n) в худшем случае.

**Память.** O(1) дополнительно (in-place).

**Устойчивость.** Нет.

### Индексация кучи в массиве

Для узла с индексом `i` (0-based):

- левый ребёнок: `2*i + 1`;
- правый ребёнок: `2*i + 2`;
- родитель: `(i-1)/2`.

```go
func siftDown(a []int, root, end int) {
	for {
		child := 2*root + 1
		if child > end {
			break
		}
		if child+1 <= end && a[child+1] > a[child] {
			child++
		}
		if a[root] >= a[child] {
			break
		}
		a[root], a[child] = a[child], a[root]
		root = child
	}
}

func HeapSort(a []int) {
	n := len(a)
	if n <= 1 {
		return
	}
	// построение кучи: с последнего родителя к корню
	for i := n/2 - 1; i >= 0; i-- {
		siftDown(a, i, n-1)
	}
	for end := n - 1; end > 0; end-- {
		a[0], a[end] = a[end], a[0]
		siftDown(a, 0, end-1)
	}
}
```

**По шагам (идея на примере `[2, 1, 3]`).**

1. **Построение max-кучи:** из массива делается такое дерево в массиве, что родитель не меньше детей. После построения корень `a[0]` — максимум всего массива.
2. **Извлечение максимума:** меняем `a[0]` с последним элементом кучи; «уменьшаем кучу»; просеиваем новый корень. Повторяем: на каждом шаге очередной максимум оказывается с конца массива, куча укорачивается.
3. Когда куча длины 1 — массив отсортирован по **возрастанию** (мы вытаскивали максимумы в конец).

**Результат.** `HeapSort([]int{3, 1, 4, 1})` → `[1 1 3 4]`.

**Пример использования.**

```go
package main

import "fmt"

func siftDown(a []int, root, end int) {
	for {
		child := 2*root + 1
		if child > end {
			break
		}
		if child+1 <= end && a[child+1] > a[child] {
			child++
		}
		if a[root] >= a[child] {
			break
		}
		a[root], a[child] = a[child], a[root]
		root = child
	}
}

func HeapSort(a []int) {
	n := len(a)
	if n <= 1 {
		return
	}
	for i := n/2 - 1; i >= 0; i-- {
		siftDown(a, i, n-1)
	}
	for end := n - 1; end > 0; end-- {
		a[0], a[end] = a[end], a[0]
		siftDown(a, 0, end-1)
	}
}

func main() {
	x := []int{3, 1, 4, 1}
	HeapSort(x)
	fmt.Println(x) // [1 1 3 4]
}
```

**Связь с Go.** Тип `container/heap` реализует **мин-кучу** на пользовательском типе через интерфейс — удобно для очередей с приоритетом, не обязательно для сортировки всего массива.

## 11. Сортировка подсчётом и поразрядная

Эти методы используют **ограниченность** или **структуру** ключей и выходят за рамки нижней оценки Ω(n log n) для сравнений.

### 11.1. Сортировка подсчётом (Counting Sort)

Подходит, если ключи — целые в диапазоне `[0, K)` и K невелико относительно n.

**Время.** O(n + K), **память** O(n + K).

```go
// CountingSort сортирует неотрицательные числа в диапазоне [0, maxVal].
func CountingSort(a []int, maxVal int) {
	if len(a) <= 1 {
		return
	}
	cnt := make([]int, maxVal+1)
	for _, v := range a {
		if v < 0 || v > maxVal {
			panic("value out of range for counting sort")
		}
		cnt[v]++
	}
	idx := 0
	for v := 0; v <= maxVal; v++ {
		for cnt[v] > 0 {
			a[idx] = v
			idx++
			cnt[v]--
		}
	}
}
```

**По шагам (вход `a = [2, 0, 2, 1]`, `maxVal = 2`).** Ключи только `0, 1, 2`.

1. Создаём `cnt` длины `maxVal+1 = 3`, нули: `[0, 0, 0]`.
2. Проходим по `a` и увеличиваем счётчики: после обхода `cnt = [1, 1, 2]` (одна ноль, одна единица, две двойки).
3. Идём `v` от `0` до `2` и **столько раз**, сколько в `cnt[v]`, записываем `v` в `a` слева направо:
   - `v=0`: один раз → `a[0]=0`;
   - `v=1`: один раз → `a[1]=1`;
   - `v=2`: два раза → `a[2]=2`, `a[3]=2`.

**Результат.** Слайс становится `[0 1 2 2]`.

**Пример использования.**

```go
package main

import "fmt"

func CountingSort(a []int, maxVal int) {
	if len(a) <= 1 {
		return
	}
	cnt := make([]int, maxVal+1)
	for _, v := range a {
		if v < 0 || v > maxVal {
			panic("value out of range for counting sort")
		}
		cnt[v]++
	}
	idx := 0
	for v := 0; v <= maxVal; v++ {
		for cnt[v] > 0 {
			a[idx] = v
			idx++
			cnt[v]--
		}
	}
}

func main() {
	grades := []int{2, 0, 2, 1} // например коды оценок 0..2
	CountingSort(grades, 2)
	fmt.Println(grades) // [0 1 2 2]
}
```

**Устойчивый** вариант: второй проход по `a`, уменьшая `cnt`, кладёт элементы в **выходной** массив с конца диапазона для каждого значения — важно при сортировке объектов по ключу.

### 11.2. Поразрядная сортировка (Radix Sort)

Часто LSD (least significant digit) для целых: стабильно сортируем по каждой цифре/байту.

**Время.** O(d · (n + r)), где d — число разрядов, r — основание (корзины).

### 12.2. `sort.Slice` и `sort.SliceStable`

Сортировка по произвольному критерию:

```go
type Person struct {
	Name string
	Age  int
}

people := []Person{
	{"Alice", 30},
	{"Bob", 25},
	{"Charlie", 30},
}

// По возрастанию возраста; при равном возрасте — по имени
sort.SliceStable(people, func(i, j int) bool {
	if people[i].Age != people[j].Age {
		return people[i].Age < people[j].Age
	}
	return people[i].Name < people[j].Name
})
```

**По шагам (как читать правило сортировки).**

- Сначала сравниваем **возраст**: младше — раньше в слайсе.
- Если возраст **одинаковый** (оба 30 у Alice и Charlie) — сравниваем **имя** по алфавиту: `Alice` < `Charlie`.

**Результат после `sort.SliceStable`.** Порядок станет: Bob (25), Alice (30), Charlie (30) — у двоих с возрастом 30 сначала Alice, потому что имя лексикографически меньше.

**Пример использования (полный `main`).**

```go
package main

import (
	"fmt"
	"sort"
)

type Person struct {
	Name string
	Age  int
}

func main() {
	people := []Person{
		{"Alice", 30},
		{"Bob", 25},
		{"Charlie", 30},
	}
	sort.SliceStable(people, func(i, j int) bool {
		if people[i].Age != people[j].Age {
			return people[i].Age < people[j].Age
		}
		return people[i].Name < people[j].Name
	})
	for _, p := range people {
		fmt.Println(p.Name, p.Age)
	}
}
```

Вывод:

```text

Alice 30
Charlie 30
```

### 12.3. Интерфейс `sort.Interface`

Для переиспользования и избежания замыканий (иногда выгоднее по аллокациям):

```go
package main

import (
	"fmt"
	"sort"
)

type Person struct {
	Name string
	Age  int
}

type ByAge []Person

func (p ByAge) Len() int           { return len(p) }
func (p ByAge) Less(i, j int) bool { return p[i].Age < p[j].Age }
func (p ByAge) Swap(i, j int)      { p[i], p[j] = p[j], p[i] }

func main() {
	people := []Person{
		{"Alice", 30},
		{"Bob", 25},
		{"Charlie", 22},
	}
	sort.Sort(ByAge(people))
	for _, p := range people {
		fmt.Println(p.Name, p.Age)
	}
}
```

**Результат.** Сортировка **только по возрасту** (вторичного ключа нет): Charlie 22, Bob 25, Alice 30.

```text
Charlie 22
Bob 25
Alice 30
```

**Пример использования.** `sort.Sort` принимает значение типа, у которого есть методы `Len`, `Less`, `Swap`. Запись `sort.Sort(ByAge(people))` приводит слайс к типу `ByAge` — методы вызываются у этого типа.

### 12.4. `sort.Search` — бинарный поиск по монотонному условию

`sort.Search(n, f)` возвращает минимальный `i` в `[0, n]`, где `f(i) == true`; если такого нет — `n`. Типичный шаблон для **первой позиции ≥ x**:

```go
import "sort"

func firstGE(a []int, x int) int {
	return sort.Search(len(a), func(i int) bool {
		return a[i] >= x
	})
}
```

Работает для **отсортированного по возрастанию** `a`.

**По шагам (массив `a = [10, 20, 20, 30]`, ищем первую позицию, где значение `>= 20`).**

`sort.Search` перебирает `i` бинарно и находит **минимальное** `i`, при котором `a[i] >= 20`. Это индекс **первой** двадцатёрки: `1`.

**Результат для разных `x`.**

| x | `firstGE(a, x)` | Смысл |
|---|-----------------|--------|
| 5 | 0 | все элементы ≥ 5, первая позиция — 0 |
| 20 | 1 | первый индекс с значением ≥ 20 |
| 25 | 3 | значения 20,20 меньше 25, первый ≥25 — индекс 3 (30) |
| 99 | 4 | все элементы меньше 99 → возвращается `len(a)` |

**Пример использования.**

```go
package main

import (
	"fmt"
	"sort"
)

func firstGE(a []int, x int) int {
	return sort.Search(len(a), func(i int) bool {
		return a[i] >= x
	})
}

func main() {
	a := []int{10, 20, 20, 30}
	fmt.Println(firstGE(a, 20)) // 1
	fmt.Println(firstGE(a, 25)) // 3
	fmt.Println(firstGE(a, 99)) // 4
}
```

## 13. Поиск: линейный и бинарный

### 13.1. Линейный поиск

**Время.** O(n).

```go
func LinearSearch(a []int, x int) int {
	for i, v := range a {
		if v == x {
			return i
		}
	}
	return -1
}
```

**По шагам (массив `[7, 3, 9]`, ищем `x = 9`).**

| i | a[i] | Действие |
|---|------|----------|
| 0 | 7 | не равно 9, идём дальше |
| 1 | 3 | не равно 9 |
| 2 | 9 | нашли, возвращаем индекс 2 |

Если `x = 5`, до конца массива не найдём — функция вернёт `-1`.

**Результат.**

```text
LinearSearch([]int{7, 3, 9}, 9) → 2
LinearSearch([]int{7, 3, 9}, 5) → -1
```

**Пример использования.**

```go
package main

import "fmt"

func LinearSearch(a []int, x int) int {
	for i, v := range a {
		if v == x {
			return i
		}
	}
	return -1
}

func main() {
	data := []int{7, 3, 9}
	fmt.Println(LinearSearch(data, 9))  // 2
	fmt.Println(LinearSearch(data, 5))   // -1
}
```

Подходит для несортированных данных или очень малых n.

### 13.2. Бинарный поиск в отсортированном слайсе

Стандарт: `sort.Search` (см. выше) или ручной цикл с избежанием переполнения при `mid := lo + (hi-lo)/2`.

```go
func BinarySearchIter(a []int, x int) (found bool, index int) {
	lo, hi := 0, len(a)
	for lo < hi {
		mid := lo + (hi-lo)/2
		if a[mid] < x {
			lo = mid + 1
		} else if a[mid] > x {
			hi = mid
		} else {
			return true, mid
		}
	}
	return false, lo // lo — позиция для вставки x с сохранением порядка
}
```

**По шагам (массив уже по возрастанию: `[10, 20, 30]`, ищем `x = 20`).**

Инвариант цикла: если `x` есть в массиве, его индекс лежит в полуинтервале `[lo, hi)`.

| Итерация | lo | hi | mid | a[mid] | Сравнение с x=20 |
|----------|----|----|-----|--------|------------------|
| старт | 0 | 3 | — | — | длина 3 |
| 1 | 0 | 3 | 1 | 20 | равно → нашли, `found=true`, `index=1` |

Для `x = 25` элемента нет: цикл сойдётся к `lo == hi`, вернётся `found=false`, `index` (позиция вставки) будет `2` — между 20 и 30.

**Результат.**

```text
BinarySearchIter([10, 20, 30], 20) → found=true,  index=1
BinarySearchIter([10, 20, 30], 25) → found=false, index=2  // куда вставить 25
BinarySearchIter([10, 20, 30], 5)  → found=false, index=0
```

**Пример использования.**

```go
package main

import "fmt"

func BinarySearchIter(a []int, x int) (found bool, index int) {
	lo, hi := 0, len(a)
	for lo < hi {
		mid := lo + (hi-lo)/2
		if a[mid] < x {
			lo = mid + 1
		} else if a[mid] > x {
			hi = mid
		} else {
			return true, mid
		}
	}
	return false, lo
}

func main() {
	a := []int{10, 20, 30}
	fmt.Println(BinarySearchIter(a, 20))
	fmt.Println(BinarySearchIter(a, 25))
	fmt.Println(BinarySearchIter(a, 5))
}
```

Вывод:

```text
true 1
false 2
false 0
```

**Инвариант.** Часто поддерживают: ответ в `[lo, hi)`.

### 13.3. Нижняя и верхняя граница (дубликаты)

Для массива с повторами:

- **нижняя граница** x — первый индекс с `≥ x`;
- **верхняя** — первый с `> x`.

Через `sort.Search`:

```go
func lowerBound(a []int, x int) int {
	return sort.Search(len(a), func(i int) bool { return a[i] >= x })
}

func upperBound(a []int, x int) int {
	return sort.Search(len(a), func(i int) bool { return a[i] > x })
}

// количество вхождений x
func countX(a []int, x int) int {
	return upperBound(a, x) - lowerBound(a, x)
}
```

**По шагам (массив `[1, 2, 2, 2, 5]`, `x = 2`).**

- `lowerBound` — первый индекс, где `a[i] >= 2`: это индекс **1** (первый двойка).
- `upperBound` — первый индекс, где `a[i] > 2`: это индекс **4** (первый элемент строго больше 2 — пятёрка).
- `countX(2) = upper - lower = 4 - 1 = 3` — три двойки подряд.

**Результат.**

```text
lowerBound → 1
upperBound → 4
countX(2)  → 3
```

**Пример использования.**

```go
package main

import (
	"fmt"
	"sort"
)

func lowerBound(a []int, x int) int {
	return sort.Search(len(a), func(i int) bool { return a[i] >= x })
}

func upperBound(a []int, x int) int {
	return sort.Search(len(a), func(i int) bool { return a[i] > x })
}

func countX(a []int, x int) int {
	return upperBound(a, x) - lowerBound(a, x)
}

func main() {
	a := []int{1, 2, 2, 2, 5}
	fmt.Println(lowerBound(a, 2)) // 1
	fmt.Println(upperBound(a, 2)) // 4
	fmt.Println(countX(a, 2))     // 3
}
```

## 14. Эффективная реализация: практические советы

1. **Не изобретать сортировку в продакшене без причины.** Для общего случая — `sort` из стандартной библиотеки; он хорошо отлажен и учитывает малые n.

2. **Меньше аллокаций.** Merge sort на больших данных — один общий буфер; избегать `append` в горячем цикле без ёмкости.

3. **Кэш.** Quick sort и merge sort с хорошей локальностью часто быстрее heap sort на практике для массивов в RAM, хотя heap sort имеет лучший худший асимптотический предел по сравнениям без доп. памяти.

4. **Стабильность.** Если нужна — явно `SliceStable` или merge sort.

5. **Сравнение структур.** Избегайте дорогих полей в `Less`: предвычисляйте ключ сортировки или используйте индексы.

6. **Параллелизм.** Рекурсивные ветви merge/quick можно распараллеливать, но константы и накладные расходы `goroutine` велики; имеет смысл только на больших n и с пулом воркеров.

7. **Контейнеры.** Для k наименьших элементов часто выгоднее **куча** размера k, чем полная сортировка.

## Приложение B: как выбрать сортировку и поиск

1. **Нужна готовая сортировка «как везде»** — `sort.Ints` / `sort.Slice` / `sort.SliceStable` в зависимости от требования устойчивости.

2. **Ключи — целые в узком диапазоне** (например, оценки 0–100, возраст, bucket id) — рассмотрите **counting sort** или этап radix; асимптотика может быть линейной по n при фиксированном K.

3. **Нужна устойчивость и предсказуемое время** на произвольных сравнимых данных — **merge sort** (или `SliceStable`).

4. **Ограничена дополнительная память**, допустим O(n) нельзя — **heap sort** или in-place quick с осторожностью по худшему случаю.

5. **Очень большие данные не в RAM** — внешняя сортировка на основе **слияния отсортированных чанков** (k-way merge); в Go часто комбинируют с файлами и буферизованным I/O.

6. **Поиск**: если данные не отсортированы — сначала сортировка O(n log n) + много запросов бинарного поиска может окупиться; если один запрос — линейный поиск проще.

## Приложение C: рекурсия, глубина и безопасность стека

**Quick sort** и **merge sort** естественно записывают рекурсивно. Глубина вызовов для merge sort при делении пополам — **O(log n)**. Для quick sort при несбалансированных разбиениях глубина может достигать **O(n)** и при больших n теоретически приблизиться к лимитам стека (на практике в Go стек растущий, но глубокая рекурсия всё равно нежелательна).

**Итеративные варианты:** явный стек из пар `(lo, hi)` для quick sort; merge sort уровнями по ширине дерева рекурсии.

**Хвостовая рекурсия** в Go не оптимизируется компилятором в цикл автоматически — полагаться на это нельзя; лучше переписать вручную.

## Приложение D: тесты и бенчмарки в Go

### Табличный тест свойств

Идея: после любой сортировки результат должен быть **неубывающим** и быть **перестановкой** исходного мультимножества значений.

```go
package sorttest

import (
	"reflect"
	"slices"
	"testing"
)

func assertSortedAndPermutation(t *testing.T, orig, got []int) {
	t.Helper()
	if !slices.IsSorted(got) {
		t.Fatalf("not sorted: %v", got)
	}
	if len(orig) != len(got) {
		t.Fatalf("length mismatch")
	}
	// для int можно сравнить отсортированные копии как мультимножества
	want := slices.Clone(orig)
	slices.Sort(want)
	gg := slices.Clone(got)
	slices.Sort(gg)
	if !reflect.DeepEqual(want, gg) {
		t.Fatalf("not a permutation: orig=%v got=%v", orig, got)
	}
}
```

### Шаблон бенчмарка

```go
func BenchmarkMySort(b *testing.B) {

		x := slices.Clone(data)
```

Сравнивайте с `b.Run("stdlib", ...)` и `sort.Ints`. Используйте `-benchmem` для учёта аллокаций.

## Приложение F: IntroSort, TimSort и гибридные схемы

**IntroSort** (introspective sort): начинают как quick sort, следят за **глубиной рекурсии**; если превышен порог (например, 2⌊log₂ n⌋), переключаются на **heap sort**, чтобы гарантировать **O(n log n)** в худшем случае по времени. Такой подход используется в C++ `std::sort`.

**TimSort** (Python, Java для объектов): гибрид **merge sort** и **run detection** — на естественных «почти отсортированных» данных используют уже готовые возрастающие подотрезки (runs), сливают их; на малых кусках — вставки. Очень хорош на реальных данных.

**Реализация в Go.** Внутренний алгоритм пакета `sort` может эволюционировать между версиями языка; для курсовой **не стоит** копировать детали из исходников как «единственный истину» на экзамене — лучше понимать семейство: «гибрид + порог на m + устойчивое слияние для stable».
